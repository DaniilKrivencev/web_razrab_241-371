# Лабораторная работа №3 — Flask: аутентификация и сессии

Веб-приложение на **Flask**, демонстрирующее работу со встроенными сессиями (для счётика посещений) и библиотекой **Flask-Login** (для аутентификации пользователей и защиты маршрутов).

## 🛠 Стек

- **Backend:** Python 3, Flask 3.0, Flask-Login 0.6.3
- **Шаблоны:** Jinja2
- **Стили:** Vanilla CSS (тёмная тема, Inter + Playfair Display)
- **Тесты:** pytest

## 🗂 Структура

```
Лабораторная 3/
├── requirements.txt
├── run.bat
└── app/
    ├── app.py           # Инициализация Flask-Login, маршруты, логика входа
    ├── __init__.py
    ├── static/
    │   └── styles.css   # Стили
    ├── templates/
    │   ├── base.html    # Базовый шаблон (flash-сообщения, динамический navbar)
    │   ├── index.html   # Главная (карточки зависят от статуса входа)
    │   ├── counter.html # Счётчик посещений (session)
    │   ├── login.html   # Форма входа с чекбоксом "Запомнить меня"
    │   └── secret.html  # Секретная страница (@login_required)
    └── tests/
        └── test_app.py  # 17 тест-кейсов
```

## 🚀 Запуск

### Вариант 1 — через bat-скрипт (Windows)
```
run.bat
```

### Вариант 2 — вручную
```bash
pip install -r requirements.txt
python app/app.py
```

Приложение откроется на **http://127.0.0.1:5000**

## 👤 Тестовый пользователь

Для проверки функционала в системе заведён один тестовый пользователь:
- Строки: **Логин:** `user` | **Пароль:** `qwerty`

## 🔑 Ключевой функционал

1. **Счётчик посещений (`/counter`)**
   - Использует `flask.session`
   - Индивидуален для каждого посетителя (браузера)
2. **Аутентификация (`/login`, `/logout`)**
   - Интеграция с `Flask-Login` 
   - Вывод `flash`-сообщений разных категорий (ошибка, успех, предупреждение)
   - Поддержка чекбокса «Запомнить меня» (кука `remember_token`)
3. **Защита маршрутов (`/secret`)**
   - При попытке зайти без авторизации — редирект на вход с сохранением параметра `next`
   - После успешного входа — автоматический возврат на секретную страницу
4. **Динамический Navbar (`base.html`)**
   - Пункты «Секретная страница» и «Выйти» видны только авторизованным
   - Пункт «Войти» исчезает после успешного входа
   - Отображается имя текущего пользователя
